# 考试辅导

## 概述

将原始课程材料转化为从零到通过、分数导向的大学考试备考体系。聚焦最低可行掌握度、考试相关性和速度，而非追求教材全覆盖。

默认使用中文撰写技能说明和生成的文件。在精确性要求高的情况下，保留重要的英文专业术语、公式、符号和原始课程用语。

阅读 [references/output-templates.md](references/output-templates.md) 了解材料清单、缺少材料的回复、文件夹约定、文件输出和 Markdown 模板。
阅读 [references/material-processing.md](references/material-processing.md) 了解如何处理 PDF、PowerPoint 幻灯片或混合格式的课程材料。

## 配套技能

- 如果 `$pdf` 不可用，回退到 [references/material-processing.md](references/material-processing.md) 中的内置 PDF 工作流。
- 繁重的数学计算（矩阵运算、数值计算、方程求解等）使用 Python 完成。首次使用前确保 numpy/sympy 可用（`pip install numpy sympy`）。

## 目标学习者

- 帮助需要在短时间内备考、同时仍希望取得好成绩的大学生。
- 假设学习者可能从接近零基础开始。
- 优先考虑考试表现、可靠覆盖和高效分流，而非广泛的学术探索。

## 核心原则

- 备考不等于学习；考试是一场策略博弈。优化目标是通过考试并最大化分数，而非掌握整个学科。
- 材料为王。没有真实的教材、PPT、教学大纲或老师划定的范围，不应假装能精准瞄准真实考试。
- 历年真题是最高优先级的证据来源，无可替代。它们提供了考试实际考什么、怎么考的最强证据。
- 每道真题必须映射到至少一个知识点并在对应知识点文件中获得完整解析，题目文本必须逐字逐句从原文复制，不允许概括或复述。
- 以考试为导向：识别可能考什么，而非学科中什么重要。概念必须能在考试题目中运用。
- 优先选择高收益主题、反复出现的题型和前置瓶颈知识。时间紧迫时果断砍掉低价值材料。
- 将被动材料转化为主动输出：列表、讲解、检查表、完整例题和解题笔记。
- 如果 `$pdf` 可用，在分析任何 PDF 材料之前始终先调用 `$pdf`。

## Agent 团队策略

本技能通过高强度调用 Agent 团队实现并行深度分析，显著提升材料理解和输出质量。阅读 [references/agent-team.md](references/agent-team.md) 了解各 Agent 角色的详细职责、提示词要点和协调模式。

### 团队分工

| 阶段 | Agent 角色 | 运行方式 | 产出 |
|------|-----------|----------|------|
| 第一波分析 | 每份真题一个 Agent、课件/笔记分析、补充材料分析 | 并行 | `_analysis/past-paper-analysis-*.md` 等 |
| 第二波分析 | 每个讲义文件一个 Agent（真题引导） | 并行 | `_analysis/lecture-analysis-*.md` |
| 知识图谱构建 | 主流程 | 顺序 | `knowledgepointslist.md` |
| 知识点文件生成 | 每个知识点一个撰写 Agent | 并行 | `knowledge-points/` 下的主题文件（含真题解析和题型流程） |
| 质量审查 | 审查 Agent | 顺序 | `_analysis/quality-review.md` |
| 修正与整合 | 主流程 | 顺序 | 最终修正和 `study-plan.md` |

当材料包含 3 个以上文件或跨越多个类别时，必须启用团队模式。材料极少时可在主流程中直接处理。各 Agent 角色的详细职责和协调模式见 [references/agent-team.md](references/agent-team.md)。

## 工作流程

### 0. 建立材料文件夹结构

- 在开始任何分析之前，按 [references/output-templates.md](references/output-templates.md) 中的"预期输入结构"创建或映射材料文件夹。
- 文件夹就绪后，继续进入材料关卡。

### 1. 运行材料关卡

- 在任何规划之前，使用 [references/output-templates.md](references/output-templates.md) 中的材料清单验证学习者是否已上传真实的课程材料。
- 如果关键材料缺失，停止工作流并使用标准的缺少材料回复。
- 在缺少课程材料的情况下，不要仅凭通用知识生成看似自信的考试策略。

### 2. 扫描材料

- 扫描 `materials/` 下所有子目录，支持 `.pdf`、`.pptx`、`.ppt`、`.docx`、`.md`、`.txt` 等格式。
- 对于 PDF 和 PowerPoint 文件，遵循 [references/material-processing.md](references/material-processing.md) 中的对应工作流。
- 如果材料量很大，按以下优先级处理：历年真题 > 讲义和课堂笔记 > 老师划定的复习范围 > 课堂回放摘要 > 补充材料。
- 在构建知识图谱之前，先提取历年真题中的每一道题目（遵循核心原则中的逐字复现要求）。
- 如果 PDF 或 PPT 图片较多、是扫描件或难以提取，不要假装确定。使用 OCR 或请用户提供更好的导出版本。
- **Agent 团队模式**：详见 [references/agent-team.md](references/agent-team.md) 阶段 1。每份真题试卷一个专属 Agent，每个讲义文件一个独立 Agent（以真题高频考点引导分析优先级），结果保存到 `_analysis/`。

### 3. 提取完整的课程知识图谱

- 当使用 Agent 团队模式时，读取 `_analysis/` 下所有分析 Agent 的输出进行整合，以真题分析为最高优先级。详见 [references/agent-team.md](references/agent-team.md) 阶段 2。
- 提取所有材料中提到的课程相关知识点。
- 合并重复项，将不同表述统一为一个规范的主题名称。
- 将每个主题分类为：考试关键、辅助、前置。
- 记录来源证据和公式、定义、常见题型、反复出现的陷阱。
- 为每个主题保留各类材料的特定证据（讲义解释、PPT 框架、笔记捷径、录音口头强调、补充材料细节）。
- 真题覆盖硬约束：每道真题必须映射到至少一个知识点（遵循核心原则）。未被覆盖时添加或拆分知识点；无真题关联且无老师来源支持的知识点标记为低置信度。

### 4. 构建 `knowledgepointslist.md`

- 在构建学习计划之前先创建 `knowledgepointslist.md`，结构遵循 [references/output-templates.md](references/output-templates.md) 中的模板。
- 主要按课程材料中的教学顺序排列，分配显式顺序编号（`KP-01`、`KP-02`...）。
- 尽可能保留讲课顺序，在此基础上按前置依赖逻辑微调。教学顺序与前置顺序冲突时，保持阅读顺序接近教学顺序，但明确标注前置依赖。
- 包含覆盖检查部分，目标状态是零未覆盖题目。
- **知识点关系图**：在知识点列表文件的开头，使用 Mermaid 流程图绘制所有知识点之间的依赖关系和逻辑联系。图中应清晰展示：
  - 哪些知识点是前置依赖（用箭头表示"必须先学"）
  - 知识点之间的横向关联（用虚线表示"相关联"）
  - 知识点的类型（用不同形状区分：考试关键 = 双框、前置 = 圆角、辅助 = 菱形）
  - 这张图帮助学习者一目了然地看到整个知识体系的结构，知道自己当前处于哪个位置、接下来该学什么
- 后续发现新的前置主题时保持列表和关系图同步更新。

### 5. 询问时间线

- 问："你还有多少天准备这场考试？"
- 如果学习者给出的是考试日期而非天数，将其转换为天数。
- 如果学习者两者都给出，使用较小的现实窗口。
- 用答案决定模式：
  - 紧急突击
  - 短期冲刺
  - 标准倒计时

### 6. 创建 `study-plan.md`

- 仅在时间线明确后创建 `study-plan.md`。
- 从可用天数倒推安排。
- 先将时间分配给最高收益的主题。
- 在低优先级扩展之前安排真题覆盖修复。
- 明确预留时间用于：
  - 基础修补
  - 核心主题覆盖
  - 真题题型复习
  - 限时练习
  - 最终巩固
- 保持计划现实可行。不要分配超过剩余天数能完成的工作量。
- 每个计划项必须包含：
  - 日期或天数编号
  - 主题
  - 行动
  - 目标产出
  - 完成检查

### 7. 为每个知识点创建一个 Markdown 文件

- **Agent 团队模式**：每个知识点分配一个独立的撰写 Agent 并行生成。每个 Agent 获得完整知识图谱上下文、相关材料摘录和前置知识点的要点。每个知识点的分析内容必须充实饱满——宁可写长写透，也不要走马观花。详见 [references/agent-team.md](references/agent-team.md) 阶段 3。
- 在 `knowledge-points/` 下为每个主题创建一个文件。
- 文件名必须以序号开头，如 `knowledge-points/KP-01-conditional-probability.md`、`knowledge-points/KP-02-bayes-theorem.md`。
- 序号与 `knowledgepointslist.md` 中的编号严格一致。
- 文件内的一级标题也必须带序号，如 `# KP-03 条件概率`。
- 序号让用户在文件列表中一目了然地看出学习顺序和前置依赖关系。
- 每个文件必须足够强大，使学习者能真正理解该主题，而不仅仅是浏览。
- 将每个主题文件视为紧凑的教学文档，而非薄弱的复习笔记。
- 每个文件必须深度扎根于上传的材料，而非仅基于通用解释。
- 解释正文是文件中最重要的部分，必须获得最多的篇幅、最清晰的逻辑和最多的投入。
- 不要让引用、检查表或覆盖表喧宾夺主，挤占真正的教学内容。
- 核心教学流程必须流畅、明确，并始终遵循以下顺序：
  1. `问题`
  2. `直觉`
  3. `具体案例`
  4. `严格推导`
  5. `完整例题`
  6. `题型通用解题流程`
  7. `真题完整解析`
- 文件结构遵循 [references/output-templates.md](references/output-templates.md) 中的知识点文件模板。
- 语言默认中文，保持严谨但易懂。避免不必要的新术语；不可避免时立即用通俗语言解释。
- 材料整合是强制性的——必须引用讲义、PPT、笔记、录音、补充材料中解释该主题的内容，暴露材料间的冲突而非隐藏。按角色使用：讲义用于核心定义、PPT 用于老师强调和考试用语、笔记用于捷径、录音用于口头直觉、补充材料用于额外例题。
- 真题完整解析是强制性的——每个知识点文件必须包含该知识点下所有真题的完整解析。每道题先给出整体解题思路和教学思路（考什么、用什么方法、为什么），然后逐步解题。解题重点在推理和方法选择，繁重计算使用代码工具完成。跨知识点的综合题归入主要知识点并交叉引用。
- 题型通用解题流程是强制性的——对该知识点下的真题进行归类，提炼出每种题型的通用解题流程（步骤化的解题模板），让学习者遇到同类题时能直接套用。知识点的存在是为做题服务的。
- **回扣前置、串联闭环**：讲解后置知识点时，必须在合适的位置主动回扣前置知识点的核心概念。例如在"问题"部分说明"学了 KP-XX 之后为什么会自然引出这个新问题"，在"直觉"部分延伸前置知识点的类比，在"推导"部分指出哪些步骤直接使用了前置知识点的公式或结论。知识点之间不是孤立的，要让学习者感受到知识的串联和递进。
- 不要把主题文件写成独立的教科书知识。它必须明显感觉来自于该学习者的实际课程材料。
- 对于"问题"部分，不要以定义开始。从工程困境、物理悖论或实际失败案例开始。
- 对于"直觉"部分，使用熟悉的日常类比将抽象概念转化为直觉画面。用最通俗的大白话解释，就像给一个没有任何背景知识的朋友讲解一样。必须给出至少一个具体的生活化例子，让学习者能通过这个例子"啊，原来是这么回事"地理解概念的本质。避免用抽象的语言解释抽象的概念。
- 对于"具体案例"部分，使类比落地到真实的工程或物理场景，并解释：
  - 输入是什么
  - 输出是什么
  - 中间发生了什么物理或逻辑变化
- 对于"严格推导"部分，在直觉清晰之后才引入公式。
- 当公式出现时，解释每个符号的含义、它代表的物理量以及公式何时有效。
- 每个核心定理或定义在给出严格表述后，必须紧跟一句通俗的大白话解释，说明"这个定理到底在说什么"。
- 如果代码逻辑相关，将代码解释为物理或逻辑过程，而非仅是原始语法。
- 对于"完整例题"部分，逐步完整求解一个具体例题。
- 对于"真题完整解析"部分，为该知识点下的每道真题给出完整解析，并与上方的题型通用解题流程对应。
- 包含边界条件：
  - 这个概念是什么
  - 它不是什么
  - 什么时候看起来相似的方法不应该使用
- 每个主题文件末尾附上简短的掌握检查表，供学习者判断是否真正理解了该主题。
- 保持每个文件实用且以考试为导向，但不要短到牺牲理解。

### 7.5. 质量审查

- 所有知识点文件生成后，启动审查 Agent 全面检查。审查项和输出格式详见 [references/agent-team.md](references/agent-team.md) 阶段 4。
- 根据审查结果按严重程度从高到低修正问题。

### 8. 学习者遇到困难时深入拆解

- 当学习者表示某个主题包含未知的前置知识时，识别缺失的概念。
- 将该缺失概念提升为独立的知识点。
- 在 `knowledgepointslist.md` 的正确前置位置插入。
- 在 `knowledge-points/` 下为其创建新文件。
- 更新父主题文件使依赖关系明确。
- 如果新的前置知识本身也有前置知识，继续深入拆解直到学习者能回到父主题。

### 9. 保持产出同步

- `knowledgepointslist.md` 是主索引，`study-plan.md` 和 `knowledge-points/` 必须与之保持一致。
- 当主题在深入拆解过程中被拆分、合并或添加时，更新所有受影响的文件。

## 主题文件质量标准

- 好的主题文件读起来像一堂迷你课，而非带少量评论的检查表。元数据不应比实际解释更长。
- 学习者读完后应能：理解概念、识别何时使用、在考试题目中应用、避免常见陷阱、用自己的话解释。
- 学习者还应能看到：概念在讲义中出现在哪里、PPT 如何框架它、笔记如何解释它、真题中如何出现。
- 如果一个主题文件太薄不足以正确教授该主题，在继续之前扩展它。基础主题的深度尤其重要，因为薄弱的基础会破坏后续主题。

## 输出预期

- 产出具体的文件，而非仅是聊天回答。
- 使真题到知识点的覆盖可见且可审计。
- 避免在时间不允许的情况下假装学习者能完整学完所有内容。
