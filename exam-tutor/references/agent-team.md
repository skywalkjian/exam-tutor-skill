# Agent 团队协调

本参考文件定义考试辅导技能中各 Agent 角色的职责、提示词要点和协调模式。

## 使用时机

- 当材料包含 3 个以上文件或跨越多个类别时，必须启用团队模式。
- 当材料极少（1–2 个同类文件）时，可在主流程中直接处理。

## 阶段 1：分波材料深度分析

讲义数据量通常极大，而真题数据量小但最重要。采用分波策略：第一波快速完成真题和小体量材料分析，第二波以真题结果引导讲义深度分析。

### 第一波：真题优先

真题和课件/笔记、补充材料第一波同时启动。由于每份真题试卷都包含独立的考试信息，**每份真题文件分配一个独立的 Agent** 进行深度分析，确保逐题提取不遗漏。

| 角色 | 输入来源 | 输出文件 |
|------|----------|----------|
| 真题分析 Agent #1 | `materials/past-papers/` 中的第 1 份试卷 | `_analysis/past-paper-analysis-1.md` |
| 真题分析 Agent #2 | `materials/past-papers/` 中的第 2 份试卷 | `_analysis/past-paper-analysis-2.md` |
| ... | ... | ... |
| 课件/笔记分析 Agent | `materials/notes/` + PPT 文件 | `_analysis/slides-notes-analysis.md` |
| 补充材料分析 Agent | `materials/supplements/` | `_analysis/supplement-analysis.md` |

**真题拆分规则**：
- 每份真题试卷（每个独立的 PDF/文件）分配一个专属 Agent
- 这样确保每道题都被仔细提取，不会因为一个 Agent 处理过多试卷而遗漏细节
- 所有真题 Agent 并行运行，充分利用并行能力

### 第二波：讲义拆分并行（真题引导）

等待第一波的真题分析完成后启动。由于讲义内容通常非常丰富，**每个讲义文件分配一个独立的 Agent**，确保每份讲义都能得到充分的深度分析。

| 角色 | 输入来源 | 输出文件 |
|------|----------|----------|
| 讲义分析 Agent #1 | 讲义文件 1 | `_analysis/lecture-analysis-1.md` |
| 讲义分析 Agent #2 | 讲义文件 2 | `_analysis/lecture-analysis-2.md` |
| ... | ... | ... |

**拆分规则**：
- 每个讲义文件分配一个独立的 Agent，不再合并处理
- 讲义内容量大，合并处理容易导致分析深度不够、遗漏关键细节
- 所有讲义 Agent 并行运行，充分利用并行能力

**真题引导**：每个讲义分析 Agent 的提示词中必须包含真题分析的高频考点排名，要求 Agent 优先深入分析与高频考点相关的章节，对低频或未出现在真题中的内容进行快速扫描而非深度分析。

### 真题分析 Agent

最重要的分析角色。每份试卷由一个专属 Agent 深度分析，确保每道题都被完整提取。

**提示词要点**：
- 读取该份真题文件（PDF 先调用 `$pdf` 如可用）
- 提取每道题的：完整题目文本（**必须逐字逐句从原文复制，包括所有数值、条件、子问题，不允许概括、缩写或用自己的话复述**）、年份、题号、子题标签、分值、考查知识点、题目类型、难度
- 如果 PDF 提取的文本不完整或有乱码，必须结合页面渲染逐字核对，确保题目文本与原件一致
- 对每道题进行初步的知识点标注（例如"本题考查条件概率和贝叶斯定理"），这些标注将在阶段 2 中与正式知识点建立映射
- 标记分值分布规律和常见陷阱
- 输出高频考点排名表（此表将作为第二波讲义分析的优先级指引）

**输出结构**：

```
# 真题深度分析 — [文件名]

## 概览
- 年份/学期：
- 考试类型：
- 总题目数：

## 逐题分析
#### 题 [编号]
- 完整题目文本（逐字复现，不得概括）：
- 分值：
- 考查知识点（初步标注）：
- 题目类型：
- 难度：

## 本卷高频考点排名
| 排名 | 知识点 | 出现次数 | 涉及题目 |

## 常见陷阱
```

**汇总阶段**：所有真题 Agent 完成后，主流程合并各 `past-paper-analysis-*.md`，生成统一的高频考点排名表供第二波讲义分析使用。同时建立"题目→知识点"的初步映射，在阶段 2 构建知识图谱时正式确认。

### 讲义/教材分析 Agent（每个子批次）

每个讲义文件由一个独立 Agent 深度分析，不再合并处理，确保分析深度。

**提示词要点**：
- 接收真题分析的高频考点排名作为优先级指引
- 对与高频考点相关的章节进行深度分析：提取核心概念、定义、定理、公式、完整推导和解题示例
- 对未出现在真题中的章节进行快速扫描：仅提取标题、核心定义和章节关系
- 识别教材强调的内容（加粗、框注、反复提及）
- 记录章节间的逻辑依赖关系
- 标注哪些内容与真题高频考点直接相关

**输出结构**：

```
# 讲义/教材分析 — [文件名]

## 分析的文件
- 文件名：

## 与真题高频考点相关的深度分析
### [章节名]
- 关联的真题考点：
- 核心概念：
- 关键定义：
- 重要定理/公式：
- 解题示例：
- 教材强调内容：

## 快速扫描的章节
### [章节名]
- 核心定义：
- 章节关系：

## 章节依赖关系

## 关键公式汇总
```

### 课件/笔记分析 Agent

**提示词要点**：
- 提取幻灯片标题层次和框架结构
- 识别老师的强调模式（反复提到、特别标注、明确说"这会考"）
- 提取演讲者备注中的高价值内容
- 从课堂笔记中提取疑难点解释和学习捷径
- 保留老师原始措辞

**输出结构**：

```
# 课件与笔记分析

## PPT 框架
### [幻灯片组名称]
- 主题结构：
- 老师强调要点：
- 演讲者备注亮点：

## 课堂笔记精华
- 疑难点解释：
- 学习捷径：
- 口头强调：

## 老师关键措辞
```

### 补充材料分析 Agent

**提示词要点**：
- 提取额外的解题示例和练习题
- 整理公式表和速查内容
- 标记与主教材不同的解释角度

## 阶段 2：汇总与知识图谱构建

所有分析 Agent 完成后，主流程执行汇总：

1. 读取 `_analysis/` 下所有分析结果
2. 合并所有真题分析 Agent 的输出（`past-paper-analysis-*.md`），建立统一的"题目→知识点"映射表，确保每道真题都被至少一个知识点覆盖
3. 合并所有讲义分析 Agent 的输出（`lecture-analysis-*.md`），解决跨文件的重复和衔接
4. 交叉比对各 Agent 发现的知识点，合并重复项
5. 利用讲义分析的章节依赖关系确定学习顺序
6. 利用课件/笔记分析识别老师重点强调的内容
7. 生成 `knowledgepointslist.md`

### 整合规则

- 当不同 Agent 对同一概念有不同命名时，优先使用老师在课件中的措辞
- 当真题考点不在讲义分析中时，仍作为考试关键主题纳入
- 当讲义内容从未在真题中出现时，降低其优先级但不删除
- 讲义深度分析中标记为"与真题高频考点相关"的内容优先纳入知识图谱

## 阶段 3：并行知识点文件生成

### 分配策略

- **每个知识点分配一个独立的撰写 Agent**，确保每个知识点都能获得充分的深度和篇幅
- 所有撰写 Agent 并行运行，充分利用并行能力

### 撰写 Agent 上下文

每个撰写 Agent 必须获得：
1. 完整的 `knowledgepointslist.md`（含 Mermaid 关系图）
2. 该知识点相关的材料分析摘录（从 `_analysis/` 提取）
3. 该知识点下所有真题的完整题目文本（Agent 需在文件中完成每道题的完整解析，并归纳题型通用解题流程）
4. **该知识点的前置知识点文件**（如已生成），或前置知识点的核心要点摘要，用于回扣和闭环
5. 知识点文件输出模板（含 SKILL.md Step 7 的质量要求）

### 撰写 Agent 核心指令

- 遵循 output-templates.md 中的知识点文件模板
- 文件名必须以序号开头，如 `KP-01-topic-name.md`，文件内一级标题也带序号
- 核心教学部分是最重要的部分，投入最多篇幅，每个知识点的分析内容必须充实饱满
- "直觉"部分必须用最通俗的大白话，配合至少一个具体的生活化例子，让学习者产生"啊，原来如此"的感觉
- 严格遵循"问题→直觉→案例→推导→例题→题型流程→真题解析"教学顺序
- **题型通用解题流程**：对该知识点下的真题进行归类，提炼每种题型的步骤化解题模板，让学习者遇到同类题时能直接套用
- **真题完整解析**：每道真题先给出整体解题思路（考什么、用什么方法、为什么），然后逐步解题。解题重点在推理和方法选择，繁重计算使用代码工具完成。综合题归入主要知识点并交叉引用
- 深度扎根于材料，而非通用解释
- **回扣前置知识点**：讲解后置知识点时，必须在合适的位置主动回扣前置知识点的核心概念，形成闭环。例如在讲解"贝叶斯定理"时回扣"条件概率"的定义和直觉，让学习者感受到知识的串联和递进。具体做法：
  - 在"问题"部分说明"为什么学了 KP-XX 之后，自然会遇到这个新问题"
  - 在"直觉"部分回扣前置知识点的类比，在其基础上延伸
  - 在"严格推导"部分指出哪些步骤直接使用了前置知识点的公式或结论
  - 在文件末尾的"链接"部分明确列出前后依赖关系
- 保存到 `knowledge-points/` 目录

## 阶段 4：质量审查

所有知识点文件生成后，生成审查 Agent 进行全面检查。

### 检查项

1. **真题覆盖完整性**：每道真题是否在至少一个知识点文件中被分析
2. **材料引用准确性**：引用的材料内容是否真实存在
3. **教学逻辑连贯性**：核心教学部分是否遵循规定顺序
4. **前置依赖一致性**：依赖关系是否无矛盾、无循环
5. **深度充分性**：基础主题是否有足够深度
6. **跨文件一致性**：不同文件对同一概念的描述是否一致
7. **真题解析完整性**：每个知识点文件中是否包含该知识点下所有真题的完整解析，是否归纳了题型通用解题流程

### 审查输出

保存到 `_analysis/quality-review.md`：

```
# 质量审查报告

## 总体评估
- 知识点文件总数：
- 通过审查数：
- 需要修正数：

## 需要修正的问题
### 问题 [N]
- 严重程度：高 / 中 / 低
- 涉及文件：
- 问题描述：
- 修正建议：

## 真题覆盖验证
| 真题 | 覆盖状态 | 对应知识点文件 |

## 深度不足的文件
```

## 阶段 5：修正与整合

主流程根据审查结果：

1. 按严重程度从高到低修正问题
2. 补充深度不足的知识点文件
3. 修复真题覆盖缺口
4. 确保 `knowledgepointslist.md` 与所有知识点文件一致
5. 确保每道真题都在对应的知识点文件中有完整解析
6. 生成 `study-plan.md`
