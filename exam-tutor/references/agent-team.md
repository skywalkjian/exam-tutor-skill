# Agent 团队协调

本参考文件定义考试辅导技能中各 Agent 角色的职责、提示词要点和协调模式。

## 使用时机

- 当材料包含 3 个以上文件或跨越多个类别时，必须启用团队模式。
- 当材料极少（1–2 个同类文件）时，可在主流程中直接处理。

## 阶段 1：分波材料深度分析

讲义数据量通常极大，而真题数据量小但最重要。采用分波策略：第一波快速完成真题和小体量材料分析，第二波以真题结果引导讲义深度分析。

### 第一波：真题优先

真题、课件/笔记和补充材料通常数据量较小，第一波同时启动。真题分析应最先完成。

| 角色 | 输入来源 | 输出文件 |
|------|----------|----------|
| 真题分析 Agent | `materials/past-papers/` | `_analysis/past-paper-analysis.md` |
| 课件/笔记分析 Agent | `materials/notes/` + PPT 文件 | `_analysis/slides-notes-analysis.md` |
| 补充材料分析 Agent | `materials/supplements/` | `_analysis/supplement-analysis.md` |

### 第二波：讲义拆分并行（真题引导）

等待第一波的真题分析完成后启动。将 `materials/lectures/` 中的文件均分给多个讲义分析 Agent。

| 角色 | 输入来源 | 输出文件 |
|------|----------|----------|
| 讲义分析 Agent #1 | 讲义文件子集 1 | `_analysis/lecture-analysis-1.md` |
| 讲义分析 Agent #2 | 讲义文件子集 2 | `_analysis/lecture-analysis-2.md` |
| ... | ... | ... |

**拆分规则**：
- 每个 Agent 处理 2–4 个讲义文件
- 文件极多时增加 Agent 数量
- 按文件自然顺序分组，相邻章节尽量分给同一 Agent

**真题引导**：每个讲义分析 Agent 的提示词中必须包含真题分析的高频考点排名，要求 Agent 优先深入分析与高频考点相关的章节，对低频或未出现在真题中的内容进行快速扫描而非深度分析。

### 真题分析 Agent

最重要的分析角色。其输出锚定后续所有分析的优先级。

**提示词要点**：
- 逐一读取每份真题文件（PDF 先调用 `$pdf` 如可用）
- 提取每道题的：完整题目文本、年份、题号、子题标签、分值、考查知识点、题目类型、难度
- 识别跨年份重复出题模式和高频考点
- 标记分值分布规律和常见陷阱
- 输出高频考点排名表（此表将作为第二波讲义分析的优先级指引）

**输出结构**：

```
# 真题深度分析

## 概览
- 覆盖年份：
- 总题目数：

## 逐题分析
### [年份] [考试名称]
#### 题 [编号]
- 完整题目文本：
- 分值：
- 考查知识点：
- 题目类型：
- 难度：

## 高频考点排名
| 排名 | 知识点 | 出现次数 | 涉及题目 |

## 重复出题模式

## 常见陷阱
```

### 讲义/教材分析 Agent（每个子批次）

**提示词要点**：
- 接收真题分析的高频考点排名作为优先级指引
- 对与高频考点相关的章节进行深度分析：提取核心概念、定义、定理、公式、完整推导和解题示例
- 对未出现在真题中的章节进行快速扫描：仅提取标题、核心定义和章节关系
- 识别教材强调的内容（加粗、框注、反复提及）
- 记录章节间的逻辑依赖关系
- 标注哪些内容与真题高频考点直接相关

**输出结构**：

```
# 讲义/教材分析（批次 N）

## 分析的文件
- 文件列表：

## 与真题高频考点相关的深度分析
### [章节名]
- 关联的真题考点：
- 核心概念：
- 关键定义：
- 重要定理/公式：
- 解题示例：
- 教材强调内容：

## 快速扫描的章节
### [章节名]
- 核心定义：
- 章节关系：

## 章节依赖关系

## 关键公式汇总
```

### 课件/笔记分析 Agent

**提示词要点**：
- 提取幻灯片标题层次和框架结构
- 识别老师的强调模式（反复提到、特别标注、明确说"这会考"）
- 提取演讲者备注中的高价值内容
- 从课堂笔记中提取疑难点解释和学习捷径
- 保留老师原始措辞

**输出结构**：

```
# 课件与笔记分析

## PPT 框架
### [幻灯片组名称]
- 主题结构：
- 老师强调要点：
- 演讲者备注亮点：

## 课堂笔记精华
- 疑难点解释：
- 学习捷径：
- 口头强调：

## 老师关键措辞
```

### 补充材料分析 Agent

**提示词要点**：
- 提取额外的解题示例和练习题
- 整理公式表和速查内容
- 标记与主教材不同的解释角度

## 阶段 2：汇总与知识图谱构建

所有分析 Agent 完成后，主流程执行汇总：

1. 读取 `_analysis/` 下所有分析结果
2. 以真题分析为最高优先级——每道真题必须被知识点覆盖
3. 合并多个讲义分析 Agent 的输出（`lecture-analysis-*.md`），解决跨批次的重复和衔接
4. 交叉比对各 Agent 发现的知识点，合并重复项
5. 利用讲义分析的章节依赖关系确定学习顺序
6. 利用课件/笔记分析识别老师重点强调的内容
7. 生成 `knowledgepointslist.md`

### 整合规则

- 当不同 Agent 对同一概念有不同命名时，优先使用老师在课件中的措辞
- 当真题考点不在讲义分析中时，仍作为考试关键主题纳入
- 当讲义内容从未在真题中出现时，降低其优先级但不删除
- 讲义深度分析中标记为"与真题高频考点相关"的内容优先纳入知识图谱

## 阶段 2.5：并行真题完整解析

知识图谱构建完成后，为每一道真题生成完整详细的解析，按知识点分类组织到 `past-paper-solutions.md`。

### 分组策略

- 将 `_analysis/past-paper-analysis.md` 中的所有题目按 `knowledgepointslist.md` 中的知识点归类
- 每个解析 Agent 负责一组相关知识点下的所有题目
- 每个 Agent 获得：
  1. 该组题目的完整题目文本
  2. 对应知识点的材料分析摘录（从 `_analysis/` 提取）
  3. `knowledgepointslist.md` 中的相关条目
  4. 解析输出模板

### 解析 Agent 提示词要点

- 对每道题进行完整的逐步解题，不允许跳步或仅给概要
- 每一步都必须说明推理依据和所用的公式/定理
- 标注得分关键步骤（漏掉会扣分的步骤）
- 标注常见错误和扣分点
- 总结同类题的解题模式
- 跨越多个知识点的综合题，归入主要知识点下并标注交叉引用

### 输出

所有解析 Agent 的结果由主流程合并为 `past-paper-solutions.md`，按知识点编号（KP-01, KP-02...）组织。

## 阶段 3：并行知识点文件生成

### 分批策略

- 每个撰写 Agent 负责 2–3 个知识点文件
- 相互依赖的知识点（前置+后续）尽量分配给同一 Agent
- 独立知识点可自由分配，尽量均匀分配工作量

### 撰写 Agent 上下文

每个撰写 Agent 必须获得：
1. 完整的 `knowledgepointslist.md`
2. 该批知识点相关的材料分析摘录（从 `_analysis/` 提取）
3. 相关真题的完整题目文本和 `past-paper-solutions.md` 中的对应完整解析
4. 知识点文件输出模板
5. SKILL.md 步骤 7 的质量要求

### 撰写 Agent 核心指令

- 遵循 output-templates.md 中的知识点文件模板
- 核心教学部分是最重要的部分，投入最多篇幅
- 严格遵循"问题→直觉→案例→推导→例题→真题"教学顺序
- 深度扎根于材料，而非通用解释
- 保存到 `knowledge-points/` 目录

## 阶段 4：质量审查

所有知识点文件生成后，生成审查 Agent 进行全面检查。

### 检查项

1. **真题覆盖完整性**：每道真题是否在至少一个知识点文件中被分析
2. **材料引用准确性**：引用的材料内容是否真实存在
3. **教学逻辑连贯性**：核心教学部分是否遵循规定顺序
4. **前置依赖一致性**：依赖关系是否无矛盾、无循环
5. **深度充分性**：基础主题是否有足够深度
6. **跨文件一致性**：不同文件对同一概念的描述是否一致
7. **真题解析完整性**：`past-paper-solutions.md` 中每道题是否有完整的逐步解题过程，是否与知识点文件中的真题分析一致

### 审查输出

保存到 `_analysis/quality-review.md`：

```
# 质量审查报告

## 总体评估
- 知识点文件总数：
- 通过审查数：
- 需要修正数：

## 需要修正的问题
### 问题 [N]
- 严重程度：高 / 中 / 低
- 涉及文件：
- 问题描述：
- 修正建议：

## 真题覆盖验证
| 真题 | 覆盖状态 | 对应知识点文件 |

## 深度不足的文件
```

## 阶段 5：修正与整合

主流程根据审查结果：

1. 按严重程度从高到低修正问题
2. 补充深度不足的知识点文件
3. 修复真题覆盖缺口
4. 确保 `knowledgepointslist.md` 与所有知识点文件一致
5. 生成 `study-plan.md`
